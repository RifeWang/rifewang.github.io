Elasticsearch Search Template

所谓 search template 搜索模板其实就是：

  1. 预先定义好查询语句 DSL 的结构并预留参数
  2. 搜索的时再传入参数值
  3. 渲染出完整的 DSL ，最后进行搜索

使用搜索模板可以将 DSL 从应用程序中解耦出来，并且可以更加灵活的更改查询语句。

例如：

GET _search/template
{
  "source" : {
    "query": {
      "match" : {
        "{{my_field}}" : "{{my_value}}"
      }
    }
  },
  "params" : {
    "my_field" : "message",
    "my_value" : "foo"
  }
}

构造出来的 DSL 就是：

{
  "query": {
    "match": {
      "message": "foo"
    }
  }
}

在模板中通过 {{ }} 的方式预留参数，然后查询时再指定对应的参数值，最后填充成具体的查询语句进行搜索。

搜索模板 API

为了实现搜索模板和查询分离，我们首先需要单独保存和管理搜索模板。

保存搜索模板

使用 scripts API 保存搜索模板（不存在则创建，存在则覆盖）。示例：

POST _scripts/<templateid>
{
  "script": {
    "lang": "mustache",
    "source": {
      "query": {
        "match": {
          "title": "{{query_string}}"
        }
      }
    }
  }
}

查询搜索模板

1

GET _scripts/<templateid>

删除搜索模板

1

DELETE _scripts/<templateid>

使用搜索模板

示例：

1
2
3
4
5
6
7

GET _search/template
{
  "id": "<templateid>",
  "params": {
    "query_string": "search words"
  }
}

params 中的参数与搜索模板中定义的一致，上文保存搜索模板的示例是 {{query_string}}，所以这里进行搜索时对应的参数就是 query_string 。

检验搜索模板

Mustache 基本语法

上文中的 {{ }} 语法其实就是 mustache language ，补充介绍下基本的语法规则。

使用 {{key}}

模板：Hello {{name}}

输入：

1
2
3

{
    "name": "Chris"
}

输出：Hello Chris

使用 {{{key}}} 避免转义

所有变量都会默认进行 HTML 转义。

模板：{{company}}

输入：

1
2
3

{
    "company": "<b>GitHub</b>"
}

输出：&lt;b&gt;GitHub&lt;/b&gt;

使用 {{{ }}} 避免转义。

模板：{{{company}}}

输入：

1
2
3

{
    "company": "<b>GitHub</b>"
}

输出：<b>GitHub</b>

使用 {{#key}} {{/key}} 构造区块

1、 当 key 是 false 或者空列表将会忽略 模板：

1
2
3
4

    Shown.
    {{#person}}
        Never shown!
    {{/person}}

输入：

1
2
3

{
    "person": false
}

输出：

1

    Shown.

2、 当 key 非空值则渲染填充 模板：

1
2
3

    {{#repo}}
        <b>{{name}}</b>
    {{/repo}}

输入：

1
2
3
4
5
6
7

{
    "repo": [
        { "name": "resque" },
        { "name": "hub" },
        { "name": "rip" }
    ]
}

输出：

1
2
3

    <b>resque</b>
    <b>hub</b>
    <b>rip</b>

3、当 key 是函数则调用后渲染 模板：

1
2
3

    {{#wrapped}}
        {{name}} is awesome.
    {{/wrapped}}

输入：

1
2
3
4
5
6
7
8

{
    "name": "Willy",
    "wrapped": function() {
        return function(text, render) {
            return "<b>" + render(text) + "</b>"
        }
    }
}

输出：

1

    <b>Willy is awesome.</b>

4、当 key 是非 false 且非列表 模板：

1
2
3

    {{#person?}}
        Hi {{name}}!
    {{/person?}}

输入：

1
2
3

{
    "person?": { "name": "Jon" }
}

输出：

1

    Hi Jon!

使用 {{^key}} {{/key}} 构造反区块

{{^key}} {{/key}} 的语法与 {{#key}} {{/key}} 类似，不同的是，当 key 不存在，或者是 false ，又或者是空列表时才渲染输出区块内容。

模板：

1
2
3
4
5
6

    {{#repo}}
        <b>{{name}}</b>
    {{/repo}}
    {{^repo}}
        No repos :(
    {{/repo}}

输入：

1
2
3

{
    "repo": []
}

输出：

1

    No repos :(

使用 {{! }} 添加注释

{{! }} 注释内容将会被忽略。

模板：

1

<h1>Today{{! ignore me }}.</h1>

输出：

1

<h1>Today.</h1>

使用 {{> }} 子模块

模板：

1
2
3
4
5
6
7
8

base.mustache:
<h2>Names</h2>
{{#names}}
    {{> user}}
{{/names}}

user.mustache:
<strong>{{name}}</strong>

其实也就等价于：

1
2
3
4

<h2>Names</h2>
{{#names}}
    <strong>{{name}}</strong>
{{/names}}

使用 {{= =}} 自定义定界符

有时候我们需要改变默认的定界符 {{ }} ，那么就可以使用 {{= =}} 的方式自定义定界符。

例如：

1

{{=<% %>=}}

定界符被定义为了 <% %>，这样原先 {{key}} 的使用方式就变成了 <%key%>。 再使用：

1

<%={{ }}=%>

就重新把定界符改回了 {{ }}。

更多语法详情请查阅官方文档 mustache language 。

结语

使用 search template 可以对搜索进行有效的解耦，即应用程序只需要关注搜索参数与返回结果，而不用关注具体使用的 DSL 查询语句，到底使用哪种 DSL 则由搜索模板进行单独管理。
